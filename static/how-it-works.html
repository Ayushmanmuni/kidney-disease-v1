System Architecture

How It Works

A detailed look at the AI pipeline — from patient data collection to real-time kidney disease risk prediction using machine learning.

📋 System Overview

Our CKD Prediction System uses a trained Random Forest Classifier to analyze clinical biomarker data and predict the probability of Chronic Kidney Disease. The system is designed for both clinical and educational use and offers two distinct input modes:

🏥

Patient Mode

A simplified 8-question form designed for individuals concerned about their kidney health. Uses medically-informed defaults for the remaining 16 parameters. Provides an easy-to-understand risk explanation.

👨‍⚕️

Doctor Mode

Full clinical interface with all 24 biomarker inputs organized in a 4-step wizard. Provides detailed health score, biomarker health grid, and feature importance chart for in-depth analysis.

Step by Step

The Prediction Pipeline

1

Data Input

The user enters clinical biomarker values via the web form. In Patient Mode, 8 key parameters are collected; the remaining 16 are filled with population-average defaults. In Doctor Mode, all 24 parameters are entered directly.

Parameters include: Age, Blood Pressure, Specific Gravity, Albumin, Sugar, Blood Glucose, Blood Urea, Serum Creatinine, Sodium, Potassium, Hemoglobin, Packed Cell Volume, WBC Count, RBC Count, and 10 categorical markers.
2

Data Preprocessing

The backend receives the 24-parameter JSON payload. Categorical values (e.g., "normal", "abnormal", "present", "not present") are encoded into numerical representations using a predefined encoding map that matches the training data format.

Encoding examples: "normal" → 1, "abnormal" → 0, "present" → 1, "notpresent" → 0, "yes" → 1, "no" → 0
3

Feature Vector Construction

The 24 encoded values are assembled into a feature vector in the exact order expected by the trained model. This ordering matches the column order from the original UCI CKD dataset used during training.

Feature order (24 features): age, bp, sg, al, su, rbc, pc, pcc, ba, bgr, bu, sc, sod, pot, hemo, pcv, wc, rc, htn, dm, cad, appet, pe, ane
4

Random Forest Prediction

The feature vector is passed to the pre-trained Random Forest Classifier. The model consists of 200 decision trees, each trained on a different bootstrap sample of the training data. Each tree "votes" independently and the final prediction is based on the majority vote.

Output: Binary classification (CKD / Not CKD) + probability score from 0% to 100% based on the proportion of trees voting for each class.
5

Result Visualization

The prediction, probability, and input data are sent back to the frontend, where results are displayed using interactive visualizations tailored to the mode:

  • Risk gauge — semicircular meter showing probability (green → yellow → orange → red)
  • Health score ring — circular score visualization (100 minus risk percentage)
  • Health grid — per-biomarker assessment with normal/warning/critical status (Doctor mode)
  • Feature importance chart — horizontal bar chart of which biomarkers most influenced the prediction (Doctor mode)
  • Patient explanation — plain-language summary of the risk level and recommended next steps (Patient mode)

🛠️ Technology Stack

Python + Flask
Backend

Lightweight Python web framework serving the API and static files. Handles prediction requests and data preprocessing.

scikit-learn
Machine Learning

Industry-standard ML library. RandomForestClassifier with 200 estimators, trained on UCI CKD dataset with 5-fold cross-validation.

Vanilla JS
Frontend

No frameworks — pure HTML5, CSS3, and JavaScript for maximum compatibility and zero dependencies. Canvas API for charts.

joblib
Model Serialization

Efficient Python object persistence. The trained model is serialized to disk and loaded on server startup for instant predictions.
